## Technical Decisions Made Tonight

### TBD Sentinel Team Pattern
For knockout bracket slots with unknown teams, we use a `TBD` sentinel team (code: "TBD", flagEmoji: "🏳️", groupStage: "KO") stored in the DB. The bracket page treats `team.code === "TBD"` as `null`, so cards render identically to the existing TBD placeholder state — but with real fixture IDs, dates, venues, and predict strips. When api-football populates the actual teams, re-running the seed replaces the TBD team IDs with real ones.

### Seed Wipe Order (FK Dependency)
The correct wipe order for a clean re-seed is:
1. `kalshiMarket.deleteMany()`
2. `liveMetric.deleteMany()`
3. `playerMatchStat.deleteMany()` ← was missing, caused RESTRICT violation
4. `match.deleteMany()`
5. `player.deleteMany()`
6. `team.deleteMany()`

### Power Rankings Formula
Correct formula (each component maxes at stated value):
```
score = ptsPerGame * 40 / 3         // max 40
      + (gdPerGame + 5) / 10 * 30   // max 30
      + gfPerGame / 4 * 20           // max 20
      + winRate * 10                 // max 10
      // total max = 100
```
The bug was `ptsPerGame * 40 / 3 * 100` — the trailing `* 100` pushed scores to ~4000+.
