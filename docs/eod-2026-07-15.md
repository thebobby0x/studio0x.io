1. `/api/matches/[id]/live` persisted clock-simulated status/elapsed + stale DB scores
   back into the DB when both feeds failed → now persists only real feed data.
2. `/api/schedule` `applyDbOverlay` refused to touch LIVE/HT rows → now max-merges DB

   group membership (→ semis labeled Group Stage MD6) → now keeps LIVE/HT scores and
   classifies by date via `classifyRound`.
4. Schedule + match detail pages now mount `LiveRefresh` (12s while live), so a stale
   server render self-corrects instead of freezing until manual reload.
