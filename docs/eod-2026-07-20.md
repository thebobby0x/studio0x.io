**BUG in Seed Clubs (Live API):** the route queries api-football with
`league=1` — which IS the World Cup — so it wrote each player's NATIONAL-TEAM
entry: club=<nation>, league="World Cup". The Leagues page then rendered
"47 clubs from 1 leagues" and "World Cup sends the most players — the clearest
argument for global supremacy" (nonsense; a CONTENT TRUTH violation live in
prod).

**Fixed tonight (display truth):** new `lib/clubData.ts REAL_CLUB_WHERE`
excludes league="World Cup"/empty from all club/league surfaces (leagues page,
ClubWCImpact, ClubContributionIndex). Now those surfaces honestly show "club
data pending" instead of a false claim. TournamentBoards (Golden Boot etc.) is
unaffected — it doesn't use club data.
