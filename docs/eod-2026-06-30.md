# EOD Report — 2026-06-30

## Session Summary

Long debugging session. The user reported five broken areas; a full audit traced
them to root causes, and a series of fixes shipped. The headline finding: **TTS
audio and anthem imports were both failing because the Vercel Blob store hit its
1 GB Hobby quota** — every Blob write was being rejected. This was masked for
hours by unrelated-looking symptoms (504 timeouts, "Audio unavailable", a missing
API-key red herring) and by Hobby's 1-hour log retention.

## THE root cause (and why it took so long)

**`[TTS] Blob put failed: Vercel Blob: Storage quota exceeded for Hobby plan (1GB maximum)`**

- The Blob store sat at **1040 MB / 1024 MB**. Every write (TTS clips *and* anthem
  imports) was rejected. Reads kept working (public URLs need no token), which made
  it look like the system was "mostly fine."
- The bloat was **not** caches (only 2.6 MB) — it was **~1 GB of orphaned duplicate
  `anthems/` files**. Every prior reimport wrote new `${Date.now()}-${title}.mp3`
  files **without deleting old ones**, so months of reimports piled up to 1 GB.
- Red herrings burned along the way: assumed missing `BLOB_READ_WRITE_TOKEN`
  (it existed), assumed missing `ELEVENLABS_API_KEY` (it existed, added Jun 20),
  chased 504 timeouts (real, but a *separate* problem). Hobby's 1-hour log
  retention meant we couldn't see the real error until the user pulled it live.

### Resolution sequence
1. **Free space:** the `anthems/` bloat had to be purged. The cache cleanup
   deliberately refuses to touch `anthems/` when the DB is empty (safety), so an
   explicit `purgeAnthems=CONFIRM_DRIVE_OK` mode + admin button was added.
   Safe because **Google Drive is the source of truth** (the importer re-downloads).
2. **Reimport** from Drive (chunked) → 19 team + 4 FIFA = **23 tracks**, clean,
   correct flags + titles, no dupes. (Costa Rica removed — did not qualify.)
3. **TTS** then worked immediately — recap + full-analysis audio both play.

## New tooling shipped today

- **`/api/admin/blob-cleanup`** — purges regenerable `tts/`+`deep-dives/` caches and
  orphaned anthem dupes. `?dryRun=true` previews. `?purgeAnthems=CONFIRM_DRIVE_OK`
  deletes ALL `anthems/` blobs (Drive is source).
  - Admin buttons: **"Free Up Blob Storage"** and **"Purge ALL Anthem Blobs (one-time)"**.
- **Chunked anthem reimport** — `/api/admin/batch-anthem?preset=true&offset=N&count=M`
  + `&finalize=true`. The **"Wipe + Reimport ALL Anthems"** button loops 4 chunks of
  6 then finalizes, so no single request hits the 60s Hobby function limit.
- **Stable anthem filenames** (`anthems/<code-or-slug>.mp3`) so reimports **overwrite**
  instead of accumulating — prevents the bloat from ever recurring.
- **`lib/tournament.ts`**, **`lib/anthemManifest.ts`**, **`lib/storyRefresh.ts`** —
  three new single-source-of-truth modules replacing scattered duplication.

## Hard-won lessons (see CLAUDE.md gotchas #15–#19)

1. **Hobby plan has three limits that bite, together:** 1 GB Blob, 60s functions,
   1-hour log retention. They masked each other today. **Vercel Pro lifts all three.**
2. **Blob `put()` fails silently-ish on quota** — symptoms look like wrong keys.
   Check Blob usage first when audio/anthem writes fail.
3. **Never write timestamped Blob filenames for re-importable content** — use stable
   names + `allowOverwrite` so re-runs don't accumulate.
4. **Multi-param admin URLs are fragile in browser/agent hand-offs** — the
   `&purgeAnthems=…` param got silently dropped twice. Put destructive actions
   behind **buttons** (URL baked into the fetch), not hand-typed URLs.
5. **Import-then-prune, never wipe-first.** An earlier reimport ran `deleteMany()`
   before importing and a mid-run failure emptied the hub. Always import first,
   prune stale rows only after ≥1 success.
