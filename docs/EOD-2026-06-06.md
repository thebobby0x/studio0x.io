## 3. The v2.3 product-image frame system
Design language, locked with your feedback:
- **Real photo** fills a **1:1 square**; never any text baked into the photo.
- **Engine = band orientation** (instant visual sort):
  - **contentOS → horizontal accent band across the BOTTOM.**
  - **templateVault → vertical accent band down the LEFT** (name reads bottom-to-top, + grid badge).
- **Band is semi-transparent** (dark scrim + ~80% brand accent) so the photo texture
  shows through while white copy stays legible.
- **Brand accent color** per niche (agentEdge blue, eComKiller red, bookedBNB coral,
  coachKit violet); product name in **Syne** (heavy), tags in **DM Mono**.
- **Credit line** in the band: `created with contentOS · by studio0x` (camelCase tokens preserved).
- **Two FOMO elements** in the top corners:
  - **Top-right — real-count stat badge** (accent2 pill). Every number is pulled from
    the kit's own `landing` data — explicit counts where the tags state one
    (e.g. `40 prompts`, `50 prompts`, `7 emails`, `12 sessions`, `4 templates`),
    otherwise the kit's true inclusion count. **No invented sales/social-proof numbers.**
  - **Top-left — punchy FOMO tag** (accent pill, ≤4 words; e.g. `WIN THE LISTING`,
    `RECOVER LOST CARTS`, `COACH ON AUTOPILOT`).

Per-product stat + tag mapping is in the chat thread (the 24-row table); easy to
tweak — re-compositing is **free** (no new render) since the photos are cached in
the container at `/tmp/out/raw_<id>.png`.
