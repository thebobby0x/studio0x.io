## 1. The alert — triaged & resolved

- **Live scan (16 Jun):** the critical `rls_disabled_in_public` lint is **gone**. ✅
- **Direct verification:** all **13 public tables have `rls_enabled = true`**
  (profiles · products · addons · product_addons · orders · order_items · entitlements ·
  ai_agents · ai_jobs · funnels · social_campaigns · customer_connections · brands).

## 2. RLS policy audit — posture is solid
Confirmed RLS is backed by correct policies (not "on with an allow-all policy"):

| Table(s) | Read | Write |
|---|---|---|
| orders / order_items / entitlements | owner only (`auth.uid()` / matching JWT email) or admin — **no anon read** | service-role only (no public write policy) |
| profiles | self or admin; self-update **cannot escalate role** | self/admin only |
| products / addons / brands | public read of **active rows only** (`is_active OR is_admin()`) | **admin only** (`is_admin()`) — prices/products can't be altered by the public |
| product_addons | public read (mapping, no PII) | admin only |

Customer emails, download tokens, and pricing are all protected. No anonymous write path anywhere.

## 3. Fix applied today
- **`function_search_path_mutable`** → hardened `public.update_updated_at()` with
  `SET search_path = ''` (migration `harden_update_updated_at_search_path`). ✅

## 4. Remaining advisor findings — all WARN, none expose customer/payment data
1. **Leaked-password protection disabled** — dashboard toggle (Auth → Password security).
   *Already on the security trio.* Lowest-effort, do it next.
2. **`is_admin()` executable via RPC** (anon + authenticated) — low risk (only reveals the
   *caller's own* admin status). The "proper" fix (revoke EXECUTE / move to a private schema)
   **risks breaking RLS**, since policies across tables call `is_admin()`. Treat as a careful,
   tested task — not a reactive change.
3. **`product-images` bucket allows listing** — low risk (only lets someone enumerate public
   marketing-image filenames; the gated `product-assets` bucket is unaffected). Tighten by
   scoping the listing policy to admins later; may touch admin image management, so handle
   deliberately.

**None are emergencies.** The only genuinely dangerous class (open table) is gone.
