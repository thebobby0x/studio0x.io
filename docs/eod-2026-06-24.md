## Architecture Notes
- The Google Drive batch import pattern: list folder via Google Drive MCP → extract file IDs → POST to `/api/admin/batch-import-anthems` with hardcoded file IDs → Vercel downloads from Drive, uploads to Blob, upserts AudioStream
- Remote execution environment has network egress restrictions (can't call production Vercel API from within the session) — hence the browser/curl trigger approach
- Preview and production share the same Neon PostgreSQL DB — anthem imports from preview deployments affect production data
